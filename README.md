# 3logiya

Минималистичный и гибкий логгер на C++ с чистым и удобным API.

3logiya — это попытка сделать логирование простым, понятным и удобным в реальных проектах.

Без лишней магии. Просто берёшь и используешь.

---

## 🚀 Быстрый старт

```cpp
#include "logger/logger.hpp"

int main() {
    auto& logger = Logger::create()
        .to_console()
        .to_file("app.log");

    logger.info("Hello, world!");
    logger.error("Something went wrong");

    LOG_INFO("Macro works too");
    LOG_DEBUG("Debug info");

    return 0;
}
```

---

## 📝 Форматирование сообщений

Методы `info`, `error` и `debug` поддерживают форматирование через `{}` и список аргументов:

```cpp
auto& logger = Logger::create()
    .to_console()
    .to_file("app.log");

int value = 42;

logger.info("User logged in: {}", "Alice");
logger.error("Failed to load resource: {}", "config.json");
logger.debug("Value of x = {}", value);
```

Макросы также поддерживают форматирование:

```cpp
LOG_INFO("User {} has {} points", "Bob", 120);
LOG_ERROR("Could not open file: {}", "data.txt");
LOG_DEBUG("Processing item {}", 7);
```

## 🌐 Глобальный и локальный логгер

3logiya поддерживает два режима использования:

### 1️⃣ Глобальный логгер

* Создаётся через `Logger::create()`.
* Сразу становится **активным** и доступным для макросов `LOG_INFO`, `LOG_ERROR`, `LOG_DEBUG`.
* Удобно, когда нужен единый логгер на весь проект.

```cpp
auto& logger = Logger::create()
    .to_console()
    .to_file("app.log");

LOG_INFO("Hello from global logger");
```

📌 Макросы автоматически используют активный логгер — не нужно передавать его в каждый вызов.

### 2️⃣ Локальный логгер

* Создаётся как обычный объект `Logger`.
* Можно конфигурировать независимо от глобального логгера.
* Идеально для **отдельных модулей или тестов**, где нужен свой вывод или фильтр.

```cpp
Logger local_logger;
local_logger.to_console();
local_logger.info("Hello from local logger");
```

⚠️ Локальный логгер не влияет на макросы и глобальный логгер.

💡 **Вывод:**

* Используй **глобальный логгер** для быстрого проекта и макросов.
* Используй **локальные логгеры**, если нужны изолированные потоки логирования или разные фильтры/файлы.

## ⚙️ Конфигурация

```cpp
auto& logger = Logger::create()
    .to_console()
    .to_file("logs.txt")
    .filter(LogLevel::ERROR);
```

---

## 🧾 Уровни логирования

```cpp
LogLevel::INFO
LogLevel::ERROR
LogLevel::DEBUG
```

DEBUG-логи игнорируются в release-сборке (`NDEBUG`).

## 🧠 Философия проекта

* Логгер должен быть удобным в использовании
* Архитектура должна быть понятной и прозрачной
* Форматирование отделено от вывода
* Стратегии вывода легко расширяются (Sink pattern)

Проект изначально создавался как инструмент для собственных проектов, но с прицелом на чистую архитектуру и удобный API.

```

    log_event.hpp         - структура события
    log_level.hpp         - уровни логирования

    file_sink.hpp         - вывод в файл

    реализация

example/
    пример использования
```
